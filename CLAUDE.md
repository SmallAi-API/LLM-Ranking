## Project Overview

This is the Chatbot Arena Leaderboard Visualizations project - a data visualization tool that displays LLM model performance (Arena Score) vs cost in an interactive scatterplot. Data is sourced from the LMArena HuggingFace space.

## Common Commands

```bash
# Setup
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip3 install -r requirements.txt

# Update leaderboard data (requires results.pkl)
python update_leaderboard_data.py

# Find models missing pricing data
python find_missing_models.py

# Run local development server
python -m http.server 8000
# Then navigate to localhost:8000/scatterplot.html
```

## Architecture

**Data Pipeline:**
- `update_leaderboard_data.py` - Downloads leaderboard data from HuggingFace, processes it using pandas, and outputs JSON files to `data/`. Set `GENERATE_HISTORICAL_CHART = True` to use historical data instead of `results.pkl`.
- `find_missing_models.py` - Compares `results.pkl` against `scatterplot-data.json` to identify models needing pricing data.

**Frontend:**
- `scatterplot.html` - Self-contained D3.js visualization. Loads JSON config files at runtime. Must be served via HTTP server to load local JSON (opening file directly won't work).

**Data Files (in `data/`):**
- `scatterplot-data.json` - Model metadata and pricing. Each model needs: `name`, `model_api_name`, `input_token_price`, `output_token_price`, `organization`, `license`, `price_source`, `model_source`. For text2img, prices are per-image instead of per-token.
- `visibility-data.json` - Controls which models show labels and label direction (left/right).
- `leaderboard-*.json` - Generated by update script; contains Arena ratings grouped by category.

**Model `model_api_name` must match the key used in the leaderboard pickle files for the scatterplot to display the model correctly.**
